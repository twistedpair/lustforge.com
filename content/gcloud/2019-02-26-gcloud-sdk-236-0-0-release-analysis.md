Google Cloud SDK 236.0.0 introduces new functionality, including flags for Cloud Asset Inventory and Cloud Dataproc, and `debug()` support for the Cloud Firestore Emulator. It promotes `services vpc-peerings` to GA and improves error handling for `gcloud auth revoke`. A minor breaking change removes the `auto-create-routes` column from `gcloud beta compute networks peerings list` output.

## Breaking Changes

- Removed `auto-create-routes` column from default output of `gcloud beta compute networks peerings list`.

## New Features by Service

### Cloud Asset Inventory

- Added `--folder` flag to `gcloud beta asset export` command.
  - Flags: `--folder`

### Cloud Dataproc

- Added `--kerberos-root-principal-password-uri`, `--kerberos-kms-key`, and `--kerberos-config-file` flags to `gcloud beta dataproc clusters create` and `gcloud beta dataproc workflow-templates set-managed-cluster`.
  - Flags: `--kerberos-root-principal-password-uri`, `--kerberos-kms-key`, `--kerberos-config-file`

### Cloud Firestore Emulator

- Released Cloud Firestore Emulator version 1.4.0 with support for the `debug(...)` function in security rules and simplified security rule evaluation and rule coverage reports.

## Credential & Auth Changes

- Modified error handling for `gcloud auth revoke` when revoking a service account token to print a friendly error message with more detailed instructions on how to revoke the token.

## API Changes

### Cloud Services

- Promoted `services vpc-peerings` command group to GA.

### Compute Engine

- Promoted `--enable-logging` and `--log-filter` flags of `gcloud compute routers nats` to beta.
