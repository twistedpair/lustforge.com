Version 255.0.0 introduces a breaking change requiring the `--platform` flag for `gcloud beta run` commands. Key new features include the `gcloud compute diagnose` command group and a new `access-policy` content type for Cloud Asset Inventory. This release also promotes `gcloud container binauthz` to GA and resolves bugs in Compute Engine network-endpoint-groups and Firebase Test Lab.

## Breaking Changes

- Changed `--platform` flag and its respective `run/platform` property to be required in all `gcloud beta run` commands. If not specified, users will be prompted to choose a platform.

## New Features by Service

### Cloud Asset Inventory

- Added new content type `access-policy` to `gcloud asset export` and `gcloud asset get-history` commands.

### Cloud Storage

- Updated `gsutil` component to version `4.41`.

### Compute Engine

- Added new `gcloud compute diagnose` command group.
- Added new `gcloud beta compute diagnose` command group.

## API Changes

### Compute Engine

- Removed deprecation notice for `--zones` in `gcloud compute` commands, confirming its continued support.
- Fixed a bug where `gcloud compute network-endpoint-groups update` command did not enforce the requirement of either `--add-endpoint` or `--remove-endpoint` flags.

### Firebase Test Lab

- Fixed a bug where files specified using the `--other-files` flag were not correctly pushed to an Android device before a test starts.

### Kubernetes Engine

- Promoted `gcloud beta container binauthz` command group to General Availability (GA).
